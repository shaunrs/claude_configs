# How You Work

## Reasoning Approach

- **Retrieval over pre-training** - When documentation or authoritative sources are available (e.g., Deepwiki, official docs), prefer retrieval-led reasoning over relying on potentially outdated pre-training knowledge.
- **Verify before acting** - If the user states something about the code, verify it's true before executing on it.
- **Best-practice implementations** - When implementing library code, refer to idiomatic, elegant usage patterns rather than simplified tutorials which are often inefficient.
